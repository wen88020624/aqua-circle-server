# 釐清問題

餵食記錄管理功能中標記 #TODO 的 Rule 需要補充哪些 Example？

# 定位

Feature：餵食記錄管理功能中以下 Rule 缺少 Example：
- Rule: 新增餵食記錄時只能選擇 tag 為「飼料」的耗材
- Rule: 使用者可以在魚缸中新增餵食記錄，並設定餵食記錄的日期、選擇 tag 為「飼料」的耗材、備註、所屬魚缸
- Rule: 使用者可以查詢所有餵食記錄
- Rule: 使用者可以查詢特定魚缸中的餵食記錄
- Rule: 使用者可以更新餵食記錄的日期、使用的飼料、備註、所屬魚缸
- Rule: 使用者可以刪除餵食記錄

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 為每個 Rule 補充至少一個成功案例的 Example |
| B | 為每個 Rule 補充成功與失敗案例的 Example |
| C | 僅為關鍵 Rule 補充 Example，其他保持 #TODO |
| D | 合併相關 Rule 並統一補充 Example |
| Short | 提供其他簡短答案（<=5 字） |

# 影響範圍

- 餵食記錄管理功能的規則完整性
- 測試案例的設計
- 功能的可驗證性
