# 釐清問題

魚缸的狀態值定義為何？

# 定位

ERM：Aquarium 實體的 status 屬性
Feature：魚缸管理功能中 Rule "魚缸狀態只能為「開缸」、「穩定」、「治療」、「閒置」"

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 使用 ERM 定義：開缸、穩定、問題、關缸 |
| B | 使用 Feature 定義：開缸、穩定、治療、閒置 |
| C | 合併兩者：開缸、穩定、問題、關缸、治療、閒置 |
| D | 其他組合（請說明） |
| Short | 提供其他簡短答案（<=5 字） |

# 影響範圍

- Aquarium.status 屬性的資料型別與約束
- 魚缸管理功能的狀態驗證規則
- 所有涉及魚缸狀態查詢的功能

# 優先級

High
- High：阻礙核心功能定義或資料建模

---
# 解決記錄

- **回答**：B - 使用 Feature 定義：開缸、穩定、治療、閒置
- **更新的規格檔**：spec/erm.dbml
- **變更內容**：更新 Aquarium.status 屬性的 note，將狀態值從「開缸、穩定、問題、關缸」改為「開缸、穩定、治療、閒置」，與 Feature 定義一致
