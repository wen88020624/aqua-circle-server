# 釐清問題

所有日期欄位的格式為何？

# 定位

ERM：所有 date 或日期相關的 string 屬性（如 setup_date、purchase_date、date 等）
Feature：所有功能中使用日期的 Rule 和 Example

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | ISO 8601 格式：YYYY-MM-DD（如 2025-01-01） |
| B | 台灣格式：YYYY/MM/DD（如 2025/01/01） |
| C | Unix timestamp（數值） |
| D | 允許多種格式，系統自動轉換 |
| Short | 提供其他簡短答案（<=5 字） |

# 影響範圍

- 所有日期欄位的資料型別與驗證規則
- 日期輸入的格式驗證
- 日期查詢與排序功能

# 優先級

Medium
- Medium：影響邊界條件或測試完整性

# 解決記錄

- **回答**：A - ISO 8601 格式：YYYY-MM-DD（如 2025-01-01）
- **更新的規格檔**：spec/erm.dbml
