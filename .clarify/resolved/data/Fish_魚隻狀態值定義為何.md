# 釐清問題

魚隻/生物的狀態值定義為何？

# 定位

ERM：Fish 實體的 status 屬性（健康、生病、死亡、隔離）
Feature：生物管理功能中提到的健康程度（過胖、過瘦、正常、生病、精神異常、死亡）

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 使用 ERM 定義：健康、生病、死亡、隔離 |
| B | 使用 Feature 健康程度：過胖、過瘦、正常、生病、精神異常、死亡 |
| C | status 與健康程度分開：status 用 ERM 定義，健康程度為獨立屬性 |
| D | 合併為單一狀態屬性 |
| Short | 提供其他簡短答案（<=5 字） |

# 影響範圍

- Fish.status 屬性的資料型別與約束
- 生物管理功能的狀態驗證規則
- 是否需要新增健康程度屬性

# 優先級

High
- High：阻礙核心功能定義或資料建模

---
# 解決記錄

- **回答**：C - status 與健康程度分開：status 用 ERM 定義，健康程度為獨立屬性
- **更新的規格檔**：spec/erm.dbml
- **變更內容**：
  - 保持 Creature.status 為 ERM 定義：健康、生病、死亡、隔離
  - 新增 Creature.health_condition 屬性，值為：過胖、過瘦、正常、生病、精神異常、死亡
