# 釐清問題

餵食記錄新增時是否需要檢查飼料狀態？

# 定位

Feature：指出餵食記錄管理功能的新增規則缺乏前置條件定義

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 必須檢查飼料狀態，過期或用完的不能使用 |
| B | 僅檢查飼料是否存在，狀態不限 |
| C | 過期飼料給予警告但仍可使用 |
| D | 不檢查飼料狀態 |
| Short | 提供其他簡短答案（<=5 字）|

# 影響範圍

影響餵食記錄新增功能的安全性檢查、飼料庫存管理，以及魚隻健康風險控制

# 優先級

High
- High：阻礙核心功能定義或資料建模

---
# 解決記錄

- **回答**：C - 過期飼料給予警告但仍可使用
- **更新的規格檔**：spec/features/餵食記錄管理.feature
- **變更內容**：新增飼料狀態檢查規則，定義過期飼料會顯示警告但仍可使用，正常飼料直接成功
