# 釐清問題

新增魚隻時是否需要檢查魚缸狀態？

# 定位

Feature：指出魚隻管理功能的新增規則缺乏前置條件定義

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 魚缸必須處於運行狀態才能新增魚隻 |
| B | 所有狀態的魚缸都可以新增魚隻 |
| C | 排除關閉或維修狀態的魚缸 |
| D | 需要額外的容量檢查 |
| Short | 提供其他簡短答案（<=5 字）|

# 影響範圍

影響魚隻新增功能的前置條件檢查、魚缸容量管理，以及系統的資料一致性維護

# 優先級

High
- High：阻礙核心功能定義或資料建模

---
# 解決記錄

- **回答**：A - 魚缸必須處於運行狀態才能新增魚隻
- **更新的規格檔**：spec/features/魚隻管理.feature
- **變更內容**：新增魚缸狀態檢查規則，定義運行狀態（開缸、穩定）允許新增魚隻，非運行狀態（問題、關缸）不允許新增
