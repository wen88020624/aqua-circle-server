# 釐清問題

生物管理功能中「所屬魚缸必須存在於系統」的驗證規則為何？

# 定位

Feature：生物管理功能中 Rule "所屬魚缸必須存在於系統"（缺少 Example）

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 新增/更新生物時，若所屬魚缸不存在，則操作失敗並顯示錯誤訊息 |
| B | 新增/更新生物時，若所屬魚缸不存在，則自動建立該魚缸 |
| C | 新增/更新生物時，允許所屬魚缸不存在（延遲驗證） |
| D | 僅在更新時驗證，新增時不驗證 |
| Short | 提供其他簡短答案（<=5 字） |

# 影響範圍

- 生物管理功能的新增/更新規則
- 魚缸與生物的關聯驗證
- 錯誤處理與使用者回饋

# 優先級

High
- High：阻礙核心功能定義或資料建模

---
# 解決記錄

- **回答**：A - 新增/更新生物時，若所屬魚缸不存在，則操作失敗並顯示錯誤訊息
- **更新的規格檔**：spec/features/生物管理.feature
- **變更內容**：為 Rule "所屬魚缸必須存在於系統" 新增兩個 Example：
  - 成功案例：所屬魚缸存在於系統中，新增/更新成功
  - 失敗案例：所屬魚缸不存在於系統中，操作失敗並顯示錯誤訊息「所屬魚缸不存在於系統中」
